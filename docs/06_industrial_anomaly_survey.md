## ISARM_MAP: [Anomaly Detection Strategy] → `src/train_autoencoder.py` + `src/generate_data.py::inject_anomalies`

### 1. Introduction
> "Anomalies are patterns in data that do not conform to expected behavior... also called outliers, exceptions, peculiarities... Detection is critical in manufacturing... for predictive maintenance, quality control, safety." (Abstract)

**ISARM Context:** Workforce flight risk = organizational anomaly. "Silent quitters", "SPOFs", "burnout" = contextual anomalies.

### 2.2 Anomaly Types
> "**Point anomalies**: individual instance anomalous... **Contextual anomalies**: anomalous in specific context (e.g., high temp in winter)... **Collective anomalies**: set of instances anomalous together." (Sec 2.2)

**ISARM Injection (Sec 3 of `generate_data.py`):**
- Type A (Silent Quitter): **Contextual** — high tenure + zero entropy (impossible combo)
- Type B (SPOF): **Contextual** — rare skill + low tenure + high OT
- Type C (Burnout): **Contextual** — accelerating OT + zero leaves + crashing perf
*All are contextual — normal marginally, anomalous jointly.*

### 3.1 Reconstruction-Based Methods (Autoencoders)
> "Autoencoders learn compressed representation... Anomaly score = reconstruction error... Undercomplete autoencoders... forces learning of normal manifold." (Sec 3.1)

**ISARM:** Direct application. Bottleneck=16 forces manifold learning. MSE = anomaly score.

### 3.1.1 Training Strategies
> "**Train on normal data only** (semi-supervised)... or **train on mixed data** (unsupervised) assuming anomalies rare." (Sec 3.1.1)

**ISARM:** Unsupervised on mixed data (6% anomalies). `threshold = 95th %ile of TRAIN NORMAL errors` — semi-supervised thresholding on unsupervised model.

### 3.1.2 Threshold Selection
> "Common: (1) fixed percentile of training errors... (2) Extreme Value Theory... (3) validation set optimization." (Sec 3.1.2)

**ISARM:** Method (1) — 95th percentile of `train_errors[y_train == 0]`. Conservative, no labeled validation needed.

### 4.2 Synthetic Data Generation
> "Synthetic benchmarks... allow controlled anomaly types... **Contextual anomalies generated by violating conditional dependencies**... e.g., high tenure but low skill entropy=0." (Sec 4.2)

**ISARM Injection Logic:** Exactly this — `inject_anomalies()` surgically violates correlations learned in `generate_clean_population()`.

### 5. Evaluation Metrics
> "Precision, Recall, F1 for binary (normal vs anomaly)... AUC-ROC, AUC-PR... **Threshold-independent** metrics preferred." (Sec 5)

**ISARM:** Classification report on binary `y_val_bin = (y_val != 0)` vs `preds = (val_err > threshold)`.

### 6. Industrial Applications
> "Manufacturing: sensor streams... predictive maintenance... **Workforce analytics emerging**... attrition prediction, skill gap detection." (Sec 6.3)

**ISARM Novelty:** First application of reconstruction-based DL anomaly detection to **workforce flight risk** fused with **shopfloor RL**.

## Key Design Alignments

| Survey Recommendation | ISARM Implementation |
|----------------------|---------------------|
| Use reconstruction error | `mse = criterion(recon, x).mean(dim=1)` |
| Semi-supervised threshold | `threshold = percentile(train_normal_errors, 95)` |
| Contextual anomalies | 3 engineered types violating conditional deps |
| Unsupervised training | No labels used in autoencoder loss |
| Dense bottleneck | 16-dim latent (not sparse) |
| Evaluation on held-out | Stratified split, report on val set |
